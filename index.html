Wasteland 3

Squad-based RPG from inXile entertainment featuring challenging tactical turn-based combat

I worked on Wasteland 3 for almost a year leading up to its release. My main contribution is the perks and quirks system. I worked closely together with designers, programmers and artists to create a system that allowed the designers to implement the majority of the perks. For the remaining perks that were too specific for a general approach, I wrote custom code. In both cases I exposed the variables so it can easily be tweaked and balanced by designers down the line, without the interference of a programmer. Other areas I worked in are the combat system, vehicle and character/vehicle customisation. Towards the end of the project I worked on bugfixes in most areas of the game.

Torn

Brains Eden 2019 entry

To cap of my time as a student I participated in Brains Eden, an International gamejam for students held at the Angela Ruskin University in Cambridge. I participated with a team of 5, 3 artists and 2 programmers. The team of the gamejam was chain reaction. We decided pretty early on to go for the storytelling award. Gameplay wise are there two players, one in the normal world and one in the spiritworld. This spiritworld is represented by a mirrored version of the normal world (yes, we took some inspiration from Stranger Things). The two worlds are torn apart and the players must work together to reunite the two worlds by pulling on the chains connecting the two worlds. pulling one chain in one world disturbs the balance of the other world, so players have to work together to win. To further drive home the cooperation only a small part of the world is visible for each player, relative to the position of the other player. This is one of those games that is hard to explain. If this piqued your interest, head over to itch.io to download the game and play it yourself or you can watch the trailer below.

Torn Gameplay Trailer from Lukas Boonen on Vimeo.

What did I do? I worked on the main gameplay elements. I made the system of the chains and platforms, helped out on the rocks that fall and added the cinematics to convey the story. Our other programmer, Stijn Billiet worked mostly on the rendering code for the lighting effect and other gameplay tasks. While our Artists, Lukas Boonen, Piter Hillewaert and Annelien Heyninck created all the assets and sound.





Outta Da Way

A chaotic sidescrolling PVP endless runner

This is a project I made together with two other programmers and two artists in Unity. It is an offline multiplayer game where the goal is to survive as long as possible while trying to defeat your opponents. The various traps and powerups scattered through the procedurally generated level make for a fun and chaotic experience where two games are never truly the same.



What did I do? The main thing I worked on was the procedural generation of the level and the background. I made sure that the level was created a little bit ahead of the camera and deleted a little bit behind the camera, all while making sure the level made sense. for example making sure the highest level only starts when it is possible to reach it and spawning falling spikes straps only when the level above is active, so they are not floating in the air. The frequency of the different traps and powerups spawn can be configured in the menu if you really like (or dislike) certain items.

In order to play the game as intended you will need at least 2 controllers (maximum 4), but to get an idea of the game you can “play” it solo as well.
Download: itch.io

Interior mapping

A shader project implementing all shader pipeline stages

The goal of this project was to create shaders using every stage of the programmable shader pipeline. After playing Spider-man for the PS4 I was intrigued by the interiors of the buildings and after some research I learned they used a technique called interior mapping. My first goal was to implement a version of this.

When this was working I continued on the other pipeline stages. I made the building dynamic, allowing to select a number of floors, rooms and dimensions of the building. I added the possibility to generate it completely on the gpu from this info and a position. For the ground I used a single quad that gets tesselated and a heightmap. Finally I added a Depth of Field effect using post processing.

This project uses a render framework from Abertay University.

The source code and an executable can be found on bitbucket.

Pirates!

Arcade pirate game where you aim for the highest score

In this arcade style pirate game the goal is to achieve the highest possible score before the time runs out. Points are scored by destroying the cannons on the forts and the enemy ships on the sea. Certain powerups can be collected to give the player a slight advantage such as higher speed, lower cooldown on the cannons and healing.

This game was made using C++ and the DAE game engine which uses SDL.
The code and a build can be found in the git repository.
The sprites used for this project come from http://kenney.nl/

GOAP AI

An AI bot using Goal Oriented Action Planning

The assignment was to create a bot for the hostprogram that could survive as long as possible. One of the requirements was to research an AI topic that was not covered in class.

I researched and implemented a GOAP (Goal Oriented Action Planning) system to power my bot. The main idea behind GOAP is to give the bot a set of goals to achieve and a set of actions it can perform. The bot will then search for a path between a number of these actions to reach its goals.
The big advantage of a system like this is the possibility to expand it later on as the project grows.

The code for this project can be found in the git repository.

Zombie Steve’s Prison Break Adventure

Puzzle Platforming Game

This is a puzzle platforming game where you play as Steve and try to escape from his prison. By using the B, X and Y buttons on the controller you can toggle the objects of the according color between two states and create a path to escape.

This game uses the Overlord Engine. The overlord engine was written by the lectors of DAE and during the fourth semester I added more features to it in class. The Engine is written in C++ and uses Nvidia Physx and Fmod.

The character model and animations are from Mixamo. and the other models were made by myself (programmer-art!).

The code and a build can be found in the git repository. A controller is required to play the game.
